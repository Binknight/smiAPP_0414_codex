# 场景分析检查清单

使用这份检查清单，从 JSON、截图和 UTG 产物中稳定提取场景信息。

## 1. 场景标识

- 场景文件路径
- App 名称或业务域
- 原始用户问题或场景描述表述
- 一句话摘要

## 2. 来自 JSON 的已确认信息

- 用户试图完成的业务任务
- 被明确点名的页面、模块、城市、地址、订单、票务或产品实体
- 明确的app的特定场景下才能触发
- 明确给出的流程描述

## 3. 新建后的产品流程

- 入口页
- 中间页面
- 关键用户动作
- 页面跳转关系
- 重点关注位置

## 4. UI 与交互覆盖度

- 已确认的组件
- 已确认的文本或标签
- 已确认的导航控件
- 已确认的状态变化
- 实际可见的样式线索

## 5. 缺失信息审计

- 缺失的页面名称
- 缺失的控件名称
- 缺失的组件层级结构
- 缺失的布局或样式细节
- 缺失的路由方向
- 缺失的前置条件或数据状态

## 6. 置信度判断

将场景归类为以下之一：

- `high confidence`：信息足够定义页面、流程和开发任务
- `medium confidence`：信息足够生成带假设的暂定 spec
- `low confidence`：未知项过多，spec 必须重点说明阻塞项和开放问题
