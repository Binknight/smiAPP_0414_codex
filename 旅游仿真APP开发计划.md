# 旅游仿真 APP 开发计划

## 1. 项目目标

- 基于通用 ArkUI 模板构建一款旅游类仿真 APP
- 按 Spec 优先打通“首页 -> 搜索 -> 结果 -> 详情 -> 下单 -> 支付 -> 订单”的核心预订链路
- 所有业务数据采用 mock 方式维护，集中放在 `entry/src/main/ets/mock/`
- 最终按 `README.md` 的构建方式完成 HAP 出包

## 2. 页面规划

### 首页
- 服务类型切换 Tab
- 快捷服务宫格
- 营销 Banner
- 热门目的地与猜你喜欢

### 搜索条件页
- 城市、日期、人数、房间数、舱位偏好等结构化字段
- 常用城市快捷选择
- 搜索条件摘要与 CTA

### 搜索结果页
- 条件摘要
- 排序与筛选
- 结果卡片列表
- 空状态与推荐文案

### 产品详情页
- 核心卖点
- 位置 / 时间 / 价格 / 退改说明
- 房型 / 舱位 / 套餐列表
- 吸底预订入口

### 下单与支付页
- 已选产品摘要
- 出行人、联系人、发票偏好
- 费用明细
- 支付方式与倒计时

### 订单 / 我的 / 客服页
- 订单状态分类与快捷操作
- 行程提醒
- 会员资产、常用信息、客服入口

## 3. 数据设计

- `homeMock.ets`：首页入口、Banner、推荐目的地
- `searchMock.ets`：服务类型、默认搜索表单、筛选与排序项
- `inventoryMock.ets`：酒店、机票、火车票、门票等商品及套餐
- `orderMock.ets`：订单列表与会员信息

## 4. 交互重点

- 首页快速进入明确搜索路径
- 列表页支持排序与条件筛选，突出价格、时间和退改信息
- 详情页套餐切换后即时刷新价格与可订状态
- 下单和支付页面提供完整费用说明与状态反馈
- 订单页支持按状态查看，并保留复购入口

## 5. 实施顺序

1. 复制模板工程并校准应用名称、资源和构建配置
2. 建立模型与 mock 数据目录
3. 重写主页面，完成多页面仿真和核心预订流程
4. 校验 README 所述构建脚本，处理签名、SDK、依赖和编译报错
5. 输出可构建的 HAP 包
