# 内容消费仿真 APP 开发计划

## 产品定位

- 目标形态：内容消费与轻创作结合的仿真 APP
- 体验核心：内容优先进入、低学习成本持续浏览、单条内容闭环互动、搜索与发现补充分发、发布入口高频可达
- 视觉方向：浅底高信息密度卡片流，兼顾图文社区的信息结构和短视频产品的强互动反馈

## 页面规划

1. 首页 Feed
- 顶部频道：推荐、关注、同城、热榜
- 搜索入口：支持关键字过滤和热搜跳转
- 内容流：卡片式封面、标题、作者、标签、互动数据
- 详情承接：点击卡片进入全屏详情浮层

2. 发现 / 搜索页
- 搜索框与联想词
- 热搜榜
- 热门话题与编辑精选专题

3. 发布页
- 模拟素材选择
- 标题、描述、标签输入
- 可见范围与评论开关展示
- 发布成功后回流首页推荐

4. 消息页
- 互动消息、评论回复、粉丝动态、系统通知
- 未读态突出显示

5. 我的页
- 个人信息与数据面板
- 作品、收藏、草稿三个视图
- 草稿可回流发布页继续编辑

## Mock 数据设计

- 统一目录：`entry/src/main/ets/mock/`
- 数据拆分：
  - `FeedMock.ets`：内容流与评论
  - `DiscoverMock.ets`：发现页话题与专题
  - `SearchMock.ets`：联想词与热搜榜
  - `MessagesMock.ets`：消息中心
  - `ProfileMock.ets`：个人主页与草稿
- 原则：每组数据独立、结构清晰、可单独替换为真实接口

## 状态与交互策略

- 首页保持频道切换状态
- 内容卡片支持点赞、收藏、关注即时反馈
- 详情页展示作者信息、标签、评论区和互动操作
- 搜索页点击热搜或建议词后回流首页并带入关键字
- 发布成功后将新内容插入推荐流首位

## 构建与交付

- 先完成页面和 mock 数据接入
- 再按 README 使用 `build.ps1` 执行构建
- 逐项修复类型、资源、签名或构建链错误
- 构建成功后整理 git 分支，创建 `prod-explore-app` 分支并提交推送
