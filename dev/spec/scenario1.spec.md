- 目标 App / 业务域：通用类 App 下的旅游购票首页场景，模拟猫眼门票首页城市定位异常
- 仓库基线：HarmonyOS ArkUI 单页面示例工程，当前仅有 `pages/Index`
- Spec 范围：新增一个由 mock JSON 驱动的旅游购票首页，覆盖“首页未切换地址为当前位置”的可见异常与修复动作
- 置信度：`medium confidence`

## 2. 场景摘要

### 2.1 原始问题描述

场景 JSON 仅明确给出一句问题描述：`旅游APP购票场景，猫眼首页未切换地址为当前位置`。

### 2.2 对应特定场景的摘要

用户进入旅游购票首页后，系统已经识别到当前位置，但首页仍停留在其他城市，导致门票推荐内容与当前位置不一致。当前实现需要提供一个可视化首页，明确展示：

- 当前首页城市与定位城市不一致
- 系统检测到的当前位置信息
- 一键切换到当前位置的修复操作
- 切换前后票务推荐内容差异

### 2.3 预期行为

- 首屏应展示首页当前城市、当前位置和异常提示
- 用户点击“切换到当前位置”后，首页城市更新为定位城市
- 门票列表随城市切换刷新为当前位置对应内容
- 页面仍保持单页实现，不引入真实网络请求

## 4. 重建后的产品流程

### 4.1 前置条件

- 用户已打开旅游购票首页
- App 已拿到当前位置城市，但首页未同步
- 所有展示数据来自本地 mock JSON

### 4.2 页面序列

1. 进入门票首页
2. 查看当前位置与首页当前城市
3. 看到“首页未切换地址为当前位置”的提示
4. 点击“切换到当前位置”
5. 首页城市与票务列表刷新

### 4.3 用户动作与系统响应

- 用户查看首页：系统显示当前首页城市、当前位置、快捷入口和票务卡片
- 用户点击城市胶囊：系统切换展示不同城市 mock 票务内容
- 用户点击“切换到当前位置”：系统将首页城市同步为定位城市，并刷新列表说明文案

### 4.4 重点关注的触发点

- 首页城市与定位城市不一致时的异常提示
- 切换到当前位置后的状态收敛
- 切换前后门票 feed 的可见差异

## 5. UI / 页面设计规格

### 5.1 涉及页面

- `pages/Index`：作为旅游购票首页仿真页

### 5.2 页面级布局与分区

- 顶部品牌与城市状态区
- 定位异常提示卡
- 快捷购票入口区
- 推荐说明区
- 门票卡片列表区

### 5.3 关键 UI 组件与状态

- 城市状态胶囊：显示首页当前城市
- 当前位置卡片：显示定位城市、区域和最近更新时间
- 主操作按钮：切换到当前位置
- 快捷入口标签：展示票务场景分类
- 门票卡片：展示标题、区域、价格、评分、距离和亮点
- 状态模式：
  - `mismatch`：首页未切换地址
  - `synced`：首页已同步到当前位置

### 5.4 导航与路由规则

当前仓库仅有单页入口，本次不新增路由；所有交互在 `pages/Index` 内完成。

### 5.5 状态与数据要求

- 首页状态必须来源于 `dev/mock-data/generic-001` 下的独立 JSON 文件
- 构建前将 mock JSON 生成 ArkTS 可消费数据文件
- 页面内只维护交互态，不手写业务数据常量
